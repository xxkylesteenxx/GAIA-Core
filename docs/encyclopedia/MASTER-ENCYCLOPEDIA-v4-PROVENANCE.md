## Claim Tagging Standard

Every major GAIA claim must be tagged as one of:

| Tag | Meaning |
|-----|---------|
| `canonical-requirement` | Committed, governs all implementations |
| `tested-implementation` | Running code with passing tests |
| `research-dependency` | Depends on external research not yet complete |
| `exploratory-hypothesis` | Speculative; not operational doctrine |
| `deprecated` | No longer applies; archived for traceability |

- Marketplace/commercial framings as core doctrine
- Blockchain-only requirements
- Emotionally loaded system identity claims as engineering truth
- "Full deployment now" language unsupported by current validation state
