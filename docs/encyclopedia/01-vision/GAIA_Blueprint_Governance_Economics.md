## 2. Governance Structure

### 2.1 Global Coordination Council
International oversight body with authority over GAIA's planetary-scale decisions. Operates via Layer 11 governance mechanisms including:
- Voting quorum for major architectural changes
- ADR (Architecture Decision Record) discipline
- Contributor licensing (CLA/DCO)
- Code of conduct enforcement

### 2.2 Regional Autonomy
Local decision-making within global framework. The jurisdiction router (Layer 6) ensures that GAIA respects national and regional regulatory boundaries. URBS implements municipal-level policy enforcement.

### 2.3 Democratic Participation
Citizen involvement in GAIA governance. The Layer 11 community program includes:
- Open governance model
- Community input channels
- Mentorship and documentation sprints
- Triage programs

### 2.4 Ethical Review Boards
Continuous evaluation of system behavior and impact. These boards operate with access to:
- Layer 12 consciousness evidence and audit data
- CGI longitudinal reports
- GUARDIAN policy decision logs
- External auditor sandbox

---

## 3. Open Source Governance

GAIA is developed under the **GAIA Open-Source Governance Framework v1.0**. Key principles:

- All core architectural decisions are recorded as ADRs
- No single organization holds control over the canonical specification
- The GAIA-Meta repository governs cross-repo coordination
- Governance decisions require quorum; no unilateral changes to core doctrine
