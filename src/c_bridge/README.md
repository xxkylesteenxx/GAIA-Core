# GAIA C Bridge — QEMU / QMP Compatibility Scaffold

This directory is the integration point for optional QEMU device-model compatibility.

## Role

- Provides a C ABI shim layer between the Rust VMM control plane and QEMU/QMP.
- QEMU is **not** a trust boundary. All security-relevant decisions are made by the Rust VMM.
- This bridge is used only when device-model compatibility requires QEMU integration.

## Planned files

| File | Purpose |
|------|---------|
| `qmp_bridge.c` | QMP socket client — sends/receives JSON commands to QEMU monitor |
| `qmp_bridge.h` | C header for Rust FFI via `bindgen` or manual `extern "C"` declarations |
| `device_policy.c` | Device allowlist enforcement before delegating to QEMU device model |

## Trust model

- All QMP commands are generated by the Rust VMM, not accepted from QEMU.
- QEMU process runs in a separate sandbox (separate uid, seccomp profile, no KVM direct access).
- The Rust VMM retains exclusive ownership of `/dev/kvm` file descriptors.
